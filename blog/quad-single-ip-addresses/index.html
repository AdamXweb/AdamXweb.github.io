Quad single IP addresses

Quad single IP addresses - think 1.1.1.1, … 8.8.8.8, 9.9.9.9 Who owns them, and what are they used for?

Anyone who has tinkered around with DNS settings are familiar with a few like 1.1.1.1, 8.8.8.8 and 9.9.9.9 (Cloudflare, Google and Quad9). I’ll be including links and readouts of whois if you’re interested to find out more.

It is interesting to learn about IP addressing, and having recently completed FIT5037 at Monash, understanding BGP (Border Gateway Protocol) routing with ASN (Autonomous system numbers)

Spoiler, if you try to visit any of the below IPs, Cloudflare is the only one that will provide a webpage at 1.1.1.1

1.1.1.1#

IP Net:

1.1.1.0/24 -> 1.1.1.0 - 1.1.1.255

Registered to APNIC

Run by CloudFlare as a DNS resolver

╭────────────────────────╮
│ ASN lookup for 1.1.1.1 │
╰────────────────────────╯

 1.1.1.1 ┌PTR one.one.one.one
         ├ASN 13335 (CLOUDFLARENET, US)
         ├RNK #78  TOP 100 AS
         ├ORG APNIC and Cloudflare DNS Resolver project
         ├NET 1.1.1.0/24 (APNIC-LABS)
         ├ABU helpdesk@apnic.net
         ├ROA ✓ VALID (1 ROA found)
         ├TYP  Anycast IP   Hosting/DC
         ├GEO Hamburg, Hamburg (DE)
         ├POR Open ports: 53, 80, 443, 2083, 2086, 2087, 8080, 8443, 8880
         └REP ✓  KNOWN GOOD as "Cloudflare Public DNS"

Ports that are open suggest 53 for DNS, 80 for HTTP, 443 for HTTPS, 2083 for radsec (Authorisation protocol),

IP Net:

2.2.0.0/16 -> 2.2.0.0 - 2.15.255.255

Assigned to: Orange.

Assigned to a french telecom, Orange. Not used for any obvious public facing services.

         ├RNK #204  TOP 500 AS
         ├ORG AS3215
         ├NET 2.2.0.0/16 (FR-TELECOM-20100712)
         ├ABU gestionip.ft@orange.com

The port active on 2.2.2.2 suggests it could be SNMP (Simple Network Management Protocol) which typically is used to communicate logging and management information.

4.0.0.0/9 -> 4.0.0.0 - 4.127.255.255

4.4.4.4 doesn’t look like it is reserved for any public facing services from Lumen, but may be assigned to one of their clients.

8.8.8.8#

IP Net:

 8.8.8.0/24 -> 8.8.8.0 - 8.8.8.255

This range is allocated to Google by ARIN - Canada and USA. Google runs a public DNS resolver on 8.8.8.8

╭────────────────────────╮
│ ASN lookup for 8.8.8.8 │
╰────────────────────────╯

 8.8.8.8 ┌PTR dns.google
         ├ASN 15169 (GOOGLE, US)
         ├RNK #1818
         ├ORG Google LLC
         ├NET 8.8.8.0/24 (GOGL)
         ├ABU network-abuse@google.com
         ├ROA ✓ VALID (1 ROA found)
         ├TYP  Anycast IP   DC  Google LLC
         ├GEO London, Westminster (GB)
         ├POR Open ports: 53, 443
         └REP ✓  KNOWN GOOD as "Google Public DNS"

Interestingly the open ports are 53 for DNS and 443 which i’m assuming is for DoH but also resolves dns.google

  • BGP.tools for 8.8.8.8
  • ipinfo for 8.8.8.8

9.9.9.9#

IP Net:

9.9.9.0/24 -> 9.9.9.0- 9.9.9.255

Quad 9. Based out of Swizerland, Quad 9 runs a DNS resolver that aims to offer better privacy.

Administered by ARIN - Canada and USA

╭────────────────────────╮
│ ASN lookup for 9.9.9.9 │
╰────────────────────────╯

 9.9.9.9 ┌PTR dns9.quad9.net
         ├ASN 19281 (QUAD9-AS-1, CH)
         ├RNK #33644
         ├ORG Quad9
         ├NET 9.9.9.0/24 (CLEAN-97)
         ├ABU abuse@quad9.net
         ├ROA ✓ VALID (1 ROA found)
         ├TYP  Anycast IP
         ├GEO Tokyo, Tokyo (JP)
         ├POR Open ports: 53, 443
         └REP ✓  KNOWN GOOD as "IBM Quad9 Public DNS"

Whilst 9.9.9.9 has port 443 open, it is exclusively used for DNS. Interestingly, the DNS record for quad9.net resolves to 216.21.3.77 (Feb 2024) which is part of the prefix 216.21.2.0/23 allocated to AS715 - WoodyNet, Inc https://woodynet.net/

Extras#

0.0.0.0#

IP Net:

0.0.0.0/8 -> 0.0.0.0 - 0.255.255.255

0.0.0.0 is reserved as a current or local network.

This along with the 10.0.0.0/8 subnet are addresses allocated for private internet as outlined in RFC1918.

OPTE#

A cool project that visualises BGP and networks is called OPTE

OPTE visualises network topology and connections that exist between them that make up the internet. Networks are often clusters of IP ranges that connect to each other, and OPTE does a good job visualising this.
